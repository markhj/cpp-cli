Adding options

In this guide, we'll look at how you can add options to your subcommands.

üåø Basics

An option is attached to a subcommand with the addOption method.

Subcommand subcommand = Subcommand();
subcommand.addOption({
.id = "option-name",
// Properties of DefineOption goes here
});

All properties are specified here CppCLI::DefineOption.

üî§ Data type

If you don't specify a data type, the default will be boolean (flag).

You can expect a string with:

.dataType = CppCLI::OptionDataType::String

üî† Short and long name

The id is sort of an "internal ID", which you will use to read the value. But toward the end-user, you'll need to specify at least one of short_name or long_name.

You can specify both, if you wish. They will in any case get automtically mapped to their CppCLI::DefineOption.

subcommand.addOption({
.id = "option-name",
// ...
.short_name = 'n',
.long_name = "name",
// Properties of DefineOption goes here
});
Note
Pay attention to that short_name is a char, and must be set with single-quotes.

‚úèÔ∏è Description

You should almost always provide a description, which will be displayed to the end-user when they make an error or request --help.

subcommand.addOption({
.id = "first-name",
// ...
.description = "Your first name",
// ...
});

üî£ Default values

You can set default values using defaultValue.

See CppCLI::Value.

Boolean

subcommand.addOption({
// ...
.defaultValue = Value(false),
// ...
});

String

subcommand.addOption({
// ...
.defaultValue = Value("Default value"),
// ...
});

üò† Required values

To require a value, you set required to true.

subcommand.addOption({
// ...
.required = true,
// ...
});

If the end-user doesn't provide a value, the CppCLI::CLI instance will catch the mistake, and ask the user to correct it.

Note
This happens prior to reaching CppCLI::Subcommand handle method, meaning you won't have to manually deal with requirements. You can rely on the fact that required values are present, when the handle executes.
Warning
The above is only true for options with required=true. Those without can still appear without any content.

üìó Reading options

Often, you'll need to read the values within your Subcommand handle methods.

Boolean

Subcommand subcommand = Subcommand();
subcommand.handle([](ConsoleIO &io, const Interpretation &interpretation) {
if (interpretation.hasFlag("show-help")) {
// Do something
}
});

String

Subcommand subcommand = Subcommand();
subcommand.handle([](ConsoleIO &io, const Interpretation &interpretation) {
// Make sure to either check the std::optional, or set the
// the DefineOption to required=true.
io->emit(interpretation.getOption("name").value().value.getAsString());
});
Note
It's recommended to check CppCLI::Value closer.
